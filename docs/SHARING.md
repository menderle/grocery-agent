# Sharing this agent with someone else

The **code** is portable; the **config** is personal. The repo holds no one's
credentials (they live in the macOS Keychain and a gitignored `.env`), so cloning is safe.
A new person personalizes these, then follows `SETUP.md`.

## Personalize-before-use checklist

| What | Where | How |
|---|---|---|
| **Home store** | `config/store.json` | Set their store id + name. The agent reads this — it's no longer hardcoded. Find the id: ask the agent to "search HEB stores near <their address>". |
| **HEB login** | parked Chrome | They log into *their* HEB account (`scripts/start_parked_chrome.sh`). Never shared. |
| **Payment card** | their HEB wallet | Their own card, saved on their HEB account (or via `scripts/add_card.py`). |
| **Secrets** | `.env` | Generated fresh by `scripts/install.sh` (their own `LIST_DROP_TOKEN`). |
| **OAuth (phone access)** | Google Cloud + `.env` | Their **own** Google OAuth app (Web client; redirect `<their base>/auth/callback`); set `GOOGLE_OAUTH_CLIENT_ID/SECRET`, `OAUTH_BASE_URL`, and `OAUTH_ALLOWED_EMAILS=their@gmail.com`. See SETUP.md step 6. |
| **MCP registration** | `.mcp.json` | Generated by `scripts/install.sh` for their machine's paths. |
| **Phone tunnel** | Tailscale | Their own tailnet + Funnel URL (`scripts/setup_tailscale_funnel.sh`). Each person gets a unique Funnel URL from their own machine hostname + tailnet. |
| **Spend limits / autonomy** | `config/policy.yaml` | Their own caps and approve/auto mode. |
| **Favor (on-demand, optional)** | own Favor account + `.env` | Separate Favor signup; `FAVOR_DEFAULT_ADDRESS`; one-time `scripts/favor_persistent_login.py` (semi-automated: agent builds cart, you place — SMS). Shares the same spend limits. Skip if not wanted. |
| **Staples & prefs** | `data/staples.json`, `data/preferences.json` | Their own. |

Nothing else is person-specific. Launchd labels are generic (`com.grocery-agent.*`),
paths derive from `GROCERY_AGENT_HOME`, and `data/` is gitignored.

## The honest "is it ready to hand over?" answer

- **Works today (with a caveat):** desktop ordering on the Mac, and phone ordering in the
  **test posture** (no-auth + dry-run, no charge). The recipient could try it end-to-end.
- **Not yet ready for unattended real ordering by a non-technical friend:** two things gate
  that —
  1. **OAuth** (so the phone connector authenticates them and real orders are safe) — see
     the OAuth task. Until then, real phone ordering means the open/no-auth posture, which
     isn't safe to leave running.
  2. **Always-on host** — it only works while the Mac is awake with the parked Chrome up.
     A Mac mini (Phase 5) removes that.
- **Maintenance they'll hit:** session re-login if the parked Chrome logs out, and the
  occasional stale-hash refresh — both one-command fixes in `docs/TROUBLESHOOTING.md`.
- **Phone access needs a paid Claude plan** (Pro/Max/Team) for custom connectors, and the
  endpoint is public-but-auth-gated (Tailscale Funnel) — see TROUBLESHOOTING. Desktop
  Claude Code on their Mac needs neither.

So: shareable as a "set it up together" project for a technical friend today; a polished
"download and go" product after OAuth + the always-on host.
